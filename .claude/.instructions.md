# 📋 Instrucciones Globales — Proyecto TFG Adventure

## Proyecto

**TFG Adventure** — Aplicación web fullstack de rutas de senderismo.

- **Frontend:** React 19 + TypeScript + TailwindCSS + Vite
- **Backend:** Spring Boot 3.x + Java 17 + MySQL 8 (TiDB Cloud)
- **Autenticación:** JWT Stateless
- **Hosting:** Frontend en ProfesionalHosting, Backend en Render.com

### Base de Datos (Sabrina)

**Tablas principales:**
- `usuario` — Email único, password BCrypt
- `ruta` — Nombre, descripción, dificultad
- `favorito` — Relación usuario ↔ ruta
- `comentario` — Mensajes en rutas

**Relaciones:**
- Usuario (1) ↔ (N) Ruta (favoritos)
- Usuario (1) ↔ (N) Comentario
- Ruta (1) ↔ (N) Comentario

## Patrones Obligatorios

### Backend (Pep)

```java
// ✅ CORRECTO
@RestController
@RequestMapping("/api/rutas")
@RequiredArgsConstructor
public class RutaController {
    private final RutaService rutaService;

    @GetMapping
    public ResponseEntity<List<RutaResponse>> getRutas() {
        return ResponseEntity.ok(rutaService.getAllRutas());
    }

    @PostMapping
    public ResponseEntity<RutaResponse> createRuta(@Valid @RequestBody CrearRutaRequest request) {
        return ResponseEntity.status(HttpStatus.CREATED)
            .body(rutaService.createRuta(request));
    }
}
```

**Reglas:**
1. Controllers sin lógica, todo en Services
2. DTOs en request/response (nunca entidades JPA)
3. @Valid en todos los endpoints
4. ResponseEntity con HTTP status correcto
5. GlobalExceptionHandler para errores

### Base de Datos (Sabrina)

```java
@Entity
@Table(name = "ruta")
@Data @Builder @NoArgsConstructor @AllArgsConstructor
public class Ruta {
    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false)
    private String nombre;

    @Enumerated(EnumType.STRING)
    private Dificultad dificultad;

    @CreationTimestamp
    private LocalDateTime fechaCreacion;
}
```

**Reglas:**
1. Índices en columnas de búsqueda frecuente
2. UNIQUE constraints para valores únicos
3. Foreign keys con ON DELETE CASCADE/RESTRICT
4. Timestamps de creación/actualización
5. utf8mb4 como charset

## Flujo de Desarrollo

1. **Backend (Pep)** → Crea endpoint REST + DTOs + Service
2. **Base de Datos (Sabrina)** → Valida schema, índices, constraints
3. **Frontend (Ani)** → Crea componentes, hooks, llamadas API
4. **Documentación (Jess)** → Documenta en README, API Reference, tutoriales
