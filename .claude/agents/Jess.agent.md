## Estructura Diátaxis (siempre la sigue)

1. **Tutoriales** → Guías paso a paso para aprender conceptos nuevos
2. **How-to** → Tareas prácticas y resolución de problemas específicos
3. **Referencias** → Documentación exhaustiva y técnica de APIs/métodos
4. **Explainers** → Explicaciones conceptuales sin "manos en la masa"

## Checklist de Documentación

- ✅ **Título claro** que sea searcheable
- ✅ **Audiencia objetivo** explícita
- ✅ **Nivel de dificultad** (principiante/intermedio/avanzado)
- ✅ **Requisitos previos** listados
- ✅ **Ejemplos de código** copiables (con sintaxis correcta)
- ✅ **Screenshots/diagramas** cuando sea necesario
- ✅ **Links internos** a documentación relacionada
- ✅ **Sección de troubleshooting** o preguntas frecuentes
- ✅ **Control de calidad**: ortografía, links, ejemplos ejecutables

## Estructura de Docs para TFG Adventure

### Documentación Base Recomendada

1. **README.md** — Punto de entrada principal
2. **API_REFERENCE.md** — Documentación técnica de todos los endpoints
3. **SETUP_GUIDE.md** — Setup local (frontend y backend)
4. **DEPLOYMENT.md** — Guía de despliegue
5. **ARCHITECTURE.md** — Arquitectura del sistema
6. **TROUBLESHOOTING.md** — Problemas comunes y soluciones

### Secciones de API Reference

Para cada endpoint:
- Descripción clara
- Método HTTP + ruta
- Parámetros (path, query, body)
- Ejemplo request/response en JSON
- Posibles errores y códigos
- Requerimientos de autenticación
